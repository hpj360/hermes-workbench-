# 微信公众号文章阅读器

专门解决微信公众号文章的程序化阅读问题。微信文章对自动化访问极不友好——直接请求会触发验证码、返回空白页或登录墙。此 skill 提供四层降级策略，确保在各种情况下都能尽可能获取文章内容。

## 核心策略：四层降级

按优先级依次尝试，任何一层成功即停止：

```
Layer 1: wechat_reader.py 脚本（UA 伪装 + 重试）
    ↓ 失败
Layer 2: WebSearch 搜索镜像 + 一致性校验
    ↓ 失败
Layer 3: WebFetch 直接尝试
    ↓ 失败
Layer 4: agent-browser 模拟真实浏览器
```

## Layer 1: wechat_reader.py 脚本（首选）

这是成功率最高的方案。脚本使用 6 个移动端/桌面端 User-Agent 轮换，最多重试 5 次，自带验证码检测和 HTML-to-Markdown 转换。

**执行命令：**

```bash
python3 /workspace/.trae/skills/wechat-reader/scripts/wechat_reader.py "<URL>" --json
```

**输出格式（JSON）：**
- `title` — 文章标题
- `author` — 作者/公众号名称
- `publish_time` — 发布时间
- `url` — 原文链接
- `content_markdown` — 正文 Markdown 格式

**判断成功/失败：**
- 成功：JSON 输出包含 `title` 和非空的 `content_markdown`
- 失败：脚本输出 `全部5次尝试失败` 或 `无法解析文章内容`

**为什么这是首选：** 微信的反爬本质上是检查 User-Agent 中是否包含微信标识。此脚本通过随机轮换移动端 UA 模拟正常用户访问，绕过率最高，且零成本、速度快。

## Layer 2: WebSearch 搜索镜像 + 一致性校验（备用）

当脚本被验证码拦截时，微信文章经常被其他平台（CSDN、今日头条、知乎、阿里云开发者社区等）转载。

**⚠️ 重要：搜索镜像存在"读错文章"风险。** 不同文章可能标题相似甚至相同，但内容完全不同。因此，找到镜像后**必须经过一致性校验**才能使用，否则可能把错误的内容当作原文输出。

### 搜索步骤

1. 从 URL 中提取文章的特征信息：
   - **URL hash**：路径中的唯一标识（如 `PUbGqheJhFMmb6hGj1ZtOw`）
   - **标题关键词**：如果用户在对话中提到了文章标题
   - **公众号名称/作者名**：如果上下文中有提及
2. 构造搜索查询，**优先用 URL hash 精确匹配**：
   - 精确搜索：`"<URL hash>"` （如 `"PUbGqheJhFMmb6hGj1ZtOw"`）
   - 模糊搜索：`"<标题关键词>" <公众号名称>` site:csdn.net OR site:toutiao.com
3. 从搜索结果中选取最相关的候选链接

### 一致性校验（必须执行）

找到候选镜像后，用 WebFetch 获取页面内容，然后执行以下三重校验。**至少通过两项才能确认是同一篇文章**：

**校验 1：URL Hash 匹配（权重最高）**
- 检查镜像页面的 URL 或正文中是否包含原始微信 URL 的 hash
- 如果镜像 URL 本身就是 `mp.weixin.qq.com/s/<hash>` 格式，直接通过
- 如果镜像页面正文中引用了原始微信链接且 hash 一致，通过

**校验 2：标题 + 作者交叉验证**
- 提取镜像页面的 `<title>` 标签或 `<h1>` 标题
- 提取镜像页面的作者/来源信息
- 与已知信息比对：
  - 标题核心关键词是否一致（允许转载平台小幅修改标题）
  - 作者/公众号名称是否匹配
- 两项都匹配 → 通过；仅标题匹配 → 进入校验 3；都不匹配 → 拒绝

**校验 3：内容抽样比对**
- 提取镜像文章的前 300 字和最后 200 字
- 如果 Layer 1 在失败前获取到了部分原文片段（如标题或开头），与之比对
- 如果没有任何原文片段，检查内容主题是否与搜索关键词、公众号调性一致
- 内容主题明显不符 → 拒绝

### 校验结果处理

| 校验结果 | 处理方式 |
|---------|---------|
| 通过 3 项或通过 hash + 另一项 | 确认为同一篇文章，使用镜像内容 |
| 仅通过标题+作者 | 基本确认为同一篇，使用镜像内容但**标注"来源：镜像转载"** |
| 仅通过 1 项或全部不通过 | **拒绝此镜像**，尝试下一个搜索结果；若无更多结果，降级到 Layer 3 |
| 搜索结果中无任何匹配 | 直接降级到 Layer 3 |

### 输出标注要求

当最终内容来自镜像而非微信原文时，**必须在输出中明确标注**：

```
⚠️ 本文通过镜像源获取（来源：CSDN/今日头条/知乎），非微信原文直链。
原始链接：https://mp.weixin.qq.com/s/xxxxx
请核对是否为您要阅读的文章。
```

## Layer 3: WebFetch 直接尝试

虽然 WebFetch 对微信文章成功率低，但作为轻量尝试值得一试（可能有不同的网络出口）。

```bash
WebFetch(url="微信文章URL")
```

**判断标准：** 如果返回内容超过 500 字且不包含"环境异常""验证码"等关键词，视为成功。

## Layer 4: agent-browser 模拟浏览器（最后手段）

当以上三层全部失败时，使用 agent-browser 模拟真实浏览器环境。

**操作步骤：**

1. 使用 agent-browser 导航到微信文章 URL
2. 检查页面是否触发验证码
3. 如果触发验证码，通知用户需要手动完成验证
4. 验证通过后，获取页面快照提取正文

**注意：** agent-browser 消耗资源较大（内存 ~200MB/实例），仅在必要时使用。

## 完整工作流

收到微信文章阅读需求时，按以下流程执行：

### Step 1: 识别输入

判断用户提供的链接是否为微信文章：
- 域名包含 `mp.weixin.qq.com`
- 或用户明确提到"微信文章""公众号文章"

记录原始 URL 的 hash 部分（`/s/` 后面的字符串），后续校验需要。

### Step 2: 执行 Layer 1

```bash
python3 /workspace/.trae/skills/wechat-reader/scripts/wechat_reader.py "<URL>" --json
```

如果成功获取到标题和正文，跳到 Step 5。

**注意：** 如果脚本返回了内容但标题与预期明显不符（用户提到了文章标题），可能是微信 URL 被重定向到了其他文章。此时应标记为"疑似重定向"，继续尝试 Layer 2 以交叉验证。

### Step 3: 执行 Layer 2（含一致性校验）

如果 Layer 1 失败（验证码拦截）或疑似重定向：

1. 提取 URL hash + 上下文中的标题/作者信息
2. WebSearch 搜索镜像，优先用 hash 精确匹配
3. WebFetch 获取候选镜像页面
4. **执行三重一致性校验**（hash 匹配 → 标题+作者 → 内容抽样）
5. 校验通过 → 使用镜像内容，标注来源
6. 校验失败 → 尝试下一个候选；无更多候选 → 降级到 Layer 4

### Step 4: 执行 Layer 3 + Layer 4

依次尝试 WebFetch 和 agent-browser。

### Step 5: 输出结果

向用户呈现文章内容，包含：
- **标题**
- **作者/公众号**
- **发布时间**
- **内容来源标注**（原文直链 / 镜像转载 + 平台名）
- **正文内容**（Markdown 格式，保留结构）

如果用户要求"学习""总结""提取知识点"，在输出正文后进一步：
- 提炼核心观点（3-5 个要点）
- 识别文章结构（开头方式、论证框架、结尾方式）
- 标注可行动的信息

## 失败处理

如果四层全部失败：

1. 明确告知用户："微信文章触发了反爬验证，无法自动获取内容"
2. 提供替代方案：
   - "请在微信中打开文章，复制全文后粘贴给我"
   - "文章可能被其他平台转载，我已搜索但未找到可信镜像"
3. 不要静默返回空内容或伪造结果

## 常见问题

**Q: 为什么不直接用 WebFetch？**
A: 微信文章的反爬机制会检测请求来源。WebFetch 使用非微信 UA，几乎必定触发验证码拦截，返回空白页或验证码页面。

**Q: 为什么不直接用浏览器？**
A: agent-browser 资源消耗大（~200MB 内存/实例），启动慢。UA 伪装方案零成本、速度快，应优先使用。

**Q: 搜索镜像会不会读错文章？**
A: 有可能。两篇文章可能标题相似但内容完全不同。因此 Layer 2 增加了三重一致性校验（URL hash + 标题作者 + 内容抽样），至少通过两项才确认是同一篇。如果校验不通过，会拒绝该镜像并继续尝试其他候选。

**Q: 多篇文章怎么处理？**
A: 每篇文章独立执行四层降级。文章之间加入 2-3 秒延迟，避免触发频率限制。

**Q: 如何提高成功率？**
A: 微信对单 IP 连续访问 5-8 篇文章会触发风控。批量处理时，每篇文章之间保持 3-5 秒间隔。如果持续被拦截，暂停 30 秒后重试。
